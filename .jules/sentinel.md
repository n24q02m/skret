## 2026-05-04 - OpenBrowser Command/Flag Injection Mitigation
**Vulnerability:** The `OpenBrowser` function in `internal/auth/prompt.go` passed strings directly to `exec.CommandContext` without validating the URL scheme or using argument separators. This allowed arbitrary command/flag injection or local file traversal if an attacker could control the URL input.
**Learning:** Using `exec.Command` with user-supplied arguments is inherently dangerous. System applications like `xdg-open` or `open` evaluate arguments sequentially; strings beginning with `-` are treated as flags unless separated, and unrestricted schemes (e.g., `file://`) can execute arbitrary local files.
**Prevention:** Always validate URL input strictly against an allowed list of schemes (e.g., `http`, `https`). Use the standard `--` argument separator when passing arguments to `exec.CommandContext` on systems that support it (e.g., `linux` with `xdg-open` and `darwin` with `open`) to prevent the OS execution command from interpreting values as options.
