## 2024-04-12 - Common Aliases for Destructive Actions
**Learning:** Adding `-f` alias to `--confirm` and `--force` flags aligns with established CLI user expectations for skipping confirmation prompts on destructive or overwrite actions. This is a critical micro-UX detail to ensure command fluency.
**Action:** Always include common aliases (like `-f`) alongside custom confirmation flags to align with established user expectations.
