## 2024-05-19 - Extracted GitHub Actions Syncer Base64 Decoding
**Learning:** The GitHub Actions syncer fetched a base64 encoded public key and previously base64 decoded it *inside* the concurrent `sync.WaitGroup` worker routines for each secret individually. Base64 decoding an invariant string inside a loop per-secret scales poorly `O(N)` and introduces significant allocation and CPU overhead relative to standard operations, scaling proportionally to the amount of secrets being handled.
**Action:** Extract the invariant decoding out of concurrent loops and pass a read-only pointer to the resulting byte array (in this case `*[32]byte`) directly into the concurrent workers to safely avoid race conditions, eliminating repetitive allocations and converting the complexity from O(N) to O(1) for decoding. Added a secret deduplicator to only send the last copy of each secret, avoiding extra network requests for secrets updated multiple times in the same sync operation.
