## 2026-04-10 - Parallelizing External API Calls (GitHub Secrets)
**Learning:** External APIs (like GitHub Actions secrets) often do not support batch write operations, leading to an N+1 problem during bulk syncs where each item requires a separate HTTP roundtrip. While migrating N+1 loops inside a service is simple, doing so against a rate-limited external API requires careful concurrency control to avoid exhaustion and connection errors.
**Action:** When faced with an N+1 loop against an external API lacking batch support, use a worker pool pattern. Implement this using a semaphore channel (e.g., `make(chan struct{}, 10)`) to bound the concurrency (e.g., to 10 simultaneous requests) combined with a `sync.WaitGroup` to wait for all calls to complete and a buffered channel to collect errors safely.
