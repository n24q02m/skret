## 2024-04-26 - Extract invariant decryption and deduplicate sync payload in GitHub syncer
**Learning:** In concurrent operations handling network requests (like synchronizing to a remote provider), redundant internal parsing and computation on identical inputs severely wastes CPU and memory resources. Furthermore, submitting duplicate keys to a remote API causes multiple network round trips and concurrent race conditions if overlapping secrets exist.
**Action:** Always deduplicate incoming payload inputs by the destination key pattern (so the final value wins predictably). Additionally, aggressively search for invariant computations—such as base64 decoding a static repository public key—inside looping or concurrent constructs and extract them to O(1) before entering the loop. In Go, safe shared structures (like pointer arrays `*[32]byte`) should be passed securely to goroutines to avoid excess allocations.
